## 토론 2: 우유 카테고리 모듈화 (DS/운영/실용 3명)

### 핵심 발견 — 실용주의자가 근본 원인 특정

> **"MilkStrategy가 필요한 게 아니라, DemandClassifier 쿼리에 버그가 있다"**

```sql
-- 버그: 당일소진 고회전 상품이 sell_days에서 누락
SUM(CASE WHEN stock_qty > 0 AND sale_qty > 0 THEN 1 ELSE 0 END) as sell_days
-- 수정: sale_qty > 0이면 판매일로 인정
SUM(CASE WHEN sale_qty > 0 THEN 1 ELSE 0 END) as sell_days
```

바나나우유(일 3.5개)가 EOD 재고=0이면 sell_days에서 빠짐 → SLOW 분류 → 예측=0

### 합의된 우선순위
| P0 | DemandClassifier sell_days 쿼리 수정 | 쿼리 2줄 |
| P1 | DEMAND_PATTERN_EXEMPT_MIDS에 047 추가 | 1줄 |
| P2 | 매장별 SKU 상한 + ABC 분류 | 중기 |
| P3 | MilkDecision 클래스 (DessertDecision 파생) | P0+P1 후 재판단 |

### 데이터 근거

## 토론 3: "발주정지 외 최소 발주 유지" 구조 (DS/악마/운영 3명)

### 합의: "121개 전부"는 불가, "선별 후 최소 발주"가 현실적

| 원래 제안 | 합의안 |
|---|---|
| 121개 전부 최소 1개 | **A+B등급 25~35개만** |
| STOP 등록만 중단 | **C등급은 처음부터 미발주** |
| 전 상품 1~2개 | **B등급만 1~2개, A는 예측 기반** |

### 악마의 변호인 경고
- 121개 전부 발주 시 판매0 상품 60개 × 4주 = **86만원 폐기 손실**
- 냉장고 1칸 = 최대 50 SKU → 121개는 물리적 불가능
- 1개만 진열 시 구매 전환율 **30~40% 하락**

### 합의된 구조
```
121개 → ABC 분류
  A등급 (월 10개+): 정상 예측 발주 (~20개)
  B등급 (월 3~9개): 최소 발주 유지 (1~2개) (~15개)
  C등급 (월 0~2개): 미발주 (발주정지 등록 불필요, 자연 제외)

+ 주간 5-KPI 평가 (셀쓰루/판매속도/유통기한/회전/기회손실)
+ STOP은 자동 집행 금지 → 수동 등록만
+ 흰우유 필수 3종 화이트리스트
```
