## 전문가별 핵심 제안

### 데이터사이언티스트

**과소예측 근본 원인 3가지 가설:**
1. 재고소진(Stockout) 편향 — 재고 바닥나면 판매=0 기록 → WMA 하향 왜곡
2. 신규/성장 매장 수요 성장 미반영 — 오픈 초기 우상향 패턴을 WMA가 못 따라감
3. 카테고리별 기온 민감도 미분리 — 음료는 1도에 5~10% 변동

**Quick Win 구현안:**
- Rolling Bias: 14일 median(actual/predicted) → 예측에 곱하기 (clamp 0.7~1.5)
- 음료 기온 계수: BEVERAGE_TEMP_MULTIPLIER = {010: 1.4, 046: 1.3, 049: 1.5}
- 신규매장 ROP 상향: data_days < 30 → ROP * 1.5

### 편의점 운영 전문가

**현장 지식 핵심:**
- "시스템은 과거 평균을 보고, 현장은 오늘의 컨텍스트를 본다"
- 기회손실 감지 필요: 재고 0된 시각 이후 판매=0은 "못 판 것"이지 "안 팔린 것"이 아님
- 충동구매형 매장(46704) vs 계획구매형 매장(46513) 구분 필요
- 배송 공백일(공휴일) 자동 2~3일치 계산 필요
- 신상품 유사상품 초기값: 동일 소분류 기존 상품 판매량으로 세팅

**잘 맞는 상품 공통점:** 매일 정해진 시간 배송, 계획 구매, 날씨 영향 적음
**안 맞는 상품 공통점:** 즉석 소비재, 기온 민감, 행사 비중 높음, 신제품 교체 잦음

### ML 엔지니어

**기술적 개선안:**
1. WMA → EWMA(지수가중이동평균) 교체: adaptive alpha (CV 기반 자동 조정)
2. data_confidence = data_days/60 → sigmoid_confidence: 비선형 성숙도 곡선
3. 매장간 전이학습: 3매장 합산 글로벌 모델 → 매장별 fine-tuning
4. 피처 추가: stockout_rate_7d, stockout_days_7, promo_price_ratio
5. 사용자 수정 데이터(ord_input_id) 가중치 학습 반영
6. 담배: 예측 기반 → Par Level(보충) 방식 전환

**confidence 공식 문제:**
- 현재 선형(14일→0.23)은 과소, 시그모이드(14일→0.37)로 교체 권장
- 카테고리별 midpoint 차등: food=14, alcohol=21, tobacco=45
