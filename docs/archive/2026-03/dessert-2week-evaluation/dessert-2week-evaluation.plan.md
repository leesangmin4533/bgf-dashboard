# Plan: 디저트 2주 평가 시스템 개선

## 1. 배경 및 문제 정의

### 현재 문제
- 인기 없는 디저트 상품이 폐기 기록이 누적되는데도 계속 발주됨
- 디저트는 유행주기가 약 2주로 빠르지만, 현재 DessertDecision의 반응 속도가 느림
- 보호기간(4주) + 판단 지연(연속 2주) + 운영자 확인 대기 = **총 6주+ 소요**

### 운영 컨텍스트
- 디저트(mid_cd=014)만 대상 — 빵(012)은 소비자 연령층이 다양하여 별도 관리
- 유통기한 범위: 2일(냉장디저트) ~ 80일+(냉동디저트)로 상품마다 다름
- 3일 발주 로직은 디저트 미포함 → 무관
- 사람이 매번 유동적으로 판단하는 것은 비효율 → 자동화 필요

### 외부 연구 인사이트
- **INSEAD**: 폐기율이 높을 때 상품 문제 vs 발주량(MOQ/배수) 문제를 구분해야 함
- **Two-period 모델**: 신상품 탐색기 2주면 유통기한 3일 상품은 4~5사이클 확보 가능
- **ABC-XYZ**: 매출×예측가능성 교차분석이 업계 표준 (현재 시스템은 유통기한 기준만)
- **유럽 슈퍼마켓 실측**: OSA와 폐기는 지수적 관계, 편의식은 매출 중심 판단이 유리

## 2. 기존 시스템 분석

### DessertDecision 현재 구조
```
분류: Cat A(냉장, 2~5일) / B(상온단기, 9~20일) / C(상온장기, 20+일) / D(냉장젤리/냉동, 80+일)
생애주기: NEW(보호) → GROWTH_DECLINE(동적) → ESTABLISHED(안정)
판정: KEEP / WATCH / REDUCE_ORDER / STOP_RECOMMEND
확정: 운영자 수동 확인 또는 30일 무판매/고폐기 자동확정
```

### 현재 타임라인 (Cat A 기준)
```
도입 → [4주 보호기간: KEEP만 가능] → [GROWTH: 연속 2주 저조 후 STOP] → [운영자 확인]
        ~~~~~~~~~~~~~~~~~~~~~~~~~~~~   ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~   ~~~~~~~~~~~~~~~
        4주                             +2주 이상                       +수일~무기한
        = 최소 6주 후에야 실제 발주 차단
```

### 관련 시스템
| 시스템 | 대상 | 역할 | 디저트 관계 |
|--------|------|------|-----------|
| DessertDecision | mid_cd=014 | 유지/중단 판정 | 직접 대상 |
| NewProductSettlement | 전 카테고리 신상품 | 5-KPI 안착 판정 | 014 포함 |
| NewProductOrderService | 푸드 신상품 | 3일 분산 발주 | 014 미포함 |
| BeverageDecision | mid_cd=039~048 | 유사 구조 | 참고용 |

## 3. 변경 범위

### 변경 1: 보호기간 단축 (4주 → 2주)
- **Cat A (냉장디저트)**: 4주 → **2주** (유통기한 2~5일, 2주 = 4~7 사이클)
- **Cat B (상온단기)**: 3주 → **2주** (유통기한 9~20일, 2주 = 1~2 사이클)
- **Cat C (상온장기)**: 4주 → **유지** (유통기한 20+일, 2주로는 데이터 부족)
- **Cat D (냉장젤리/냉동)**: 4주 → **유지** (유통기한 80+일, 장기 관찰 필요)

**근거**: Two-period 모델에서 탐색기 2주면 단기유통 상품은 충분한 데이터 확보

### 변경 2: GROWTH 판단 기준 강화
- **현재**: 연속 2주 저조(`sale_rate < 30%`) → STOP_RECOMMEND
- **변경**:
  - **1주 저조**: 즉시 WATCH (경고 태그 + 대시보드 노출)
  - **2주 연속 저조**: STOP_RECOMMEND (기존과 동일하지만 2주 빨리 도달)

**Cat A 구체안**:
```
GROWTH_DECLINE 진입 (2주 후):
  sale_rate < 30% 연속 1주 → WATCH
  sale_rate < 30% 연속 2주 → STOP_RECOMMEND
  sale_rate < 50% 연속 1주 → WATCH (기존 2주에서 단축)
  sale_rate < 50% 연속 2주 → REDUCE_ORDER
```

### 변경 3: 자동 확정 기준 강화
- **현재**: 30일 무판매 → 자동 CONFIRMED_STOP
- **변경**:
  - **Cat A/B**: **14일** 무판매 → 자동 CONFIRMED_STOP
  - **Cat C**: 30일 유지
  - **Cat D**: 60일 유지
  - 상수화: `DESSERT_AUTO_CONFIRM_DAYS = {"A": 14, "B": 14, "C": 30, "D": 60}`

### 변경 4: 폐기율 기반 자동 확정 확대
- **현재**: waste_rate >= 150% + 연속 2주 → 자동 확정
- **추가**:
  - **waste_rate >= 100% + 보호기간 종료 즉시** → 자동 CONFIRMED_STOP
  - 운영자 확인 불필요 (폐기가 판매를 초과하면 즉시 손절)

```
        ~~~~~~~~~~
        = 최소 2주에 발주 차단 (기존 6주+에서 70% 단축)
```
