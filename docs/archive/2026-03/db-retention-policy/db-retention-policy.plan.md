# DB 보존 정책 (db-retention-policy)

## 1. 개요

### 문제
DB 기록이 무한히 쌓이고 있다. 보존 정책(삭제 규칙)이 없어서 매장당 DB가 매년 ~200MB씩 증가.

### 현재 상태
| 테이블 | 현재 행수 (46513) | 일 평균 증가 | 연간 예상 | 보존 정책 |
|--------|-------------------|-------------|----------|----------|
| eval_outcomes | 108,243 | ~1,900행 | ~69만행 | ❌ 없음 |
| prediction_logs | 99,379 | ~1,700행 | ~62만행 | ❌ 없음 |
| hourly_sales_detail | 83,161 | ~1,400행 | ~51만행 | ❌ 없음 |
| calibration_history | 265 | ~5행 | ~1,800행 | ❌ 없음 |

참고: 로그 파일은 30일 정리, 스크린샷은 7일 정리 — DB만 무한 성장.

### 목표
- 매장당 DB 크기를 적정 수준(~150MB)으로 유지
- 예측/학습에 필요한 최소 데이터 보존
- 매일 자동 실행 (수동 개입 불필요)

## 2. 범위

### 포함
- eval_outcomes: 90일 보존 (ML 학습 윈도우 90일 기준)
- prediction_logs: 60일 보존 (디버깅 용도)
- hourly_sales_detail: 90일 보존
- calibration_history: 120일 보존

### 제외
- daily_sales: 장기 보존 필요 (예측 기반 데이터)
- order_tracking: 감사 기록, 장기 보존
- 테이블 DROP/스키마 변경: 별도 작업
