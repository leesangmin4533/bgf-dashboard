# BGF 리테일 사이트 전체 메뉴 분석 (2026-02-27)

## 개요

BGF 리테일 점포 관리 사이트(store.bgfretail.com)의 **전체 메뉴 구조**를 탐색하여,
현재 프로젝트에서 사용 중인 8개 화면 외에 **활용 가능한 추가 화면/API**를 분석한 문서.

- **전체 메뉴**: 272개 (ds_orgMenu)
- **화면(리프 메뉴)**: ~180개
- **현재 사용 중**: 8개 (★ 표시)
- **분석 방법**: nexacro `ds_orgMenu` 데이터셋 직접 추출 + DOM 기반 서브메뉴 스캔

---

## API 엔드포인트 패턴 (탐색 시 발견)

| 정산 | `/stjs{NNN}/selSearch` | - |
| 검수전표 | `/stgj{NNN}/selSearch` | - |

| 공통 | `/search/selNoticeTotalSearch` | 공지사항 검색 |

### Direct API 전환 가능 화면

기존 `direct_api_fetcher.py` 패턴(SSV 프로토콜 + fetch())으로 전환 가능:

1. **STJK010_M0** (현재고) → `/stjk010/selSearch` → 실시간 재고 조회
2. **STGJ300_M0** (입고예정) → `/stgj300/selSearch` → 입고예정 조회
3. **STBJ330_M0** (발주정지상품) → `/stbj330/selSearch` → 발주정지 목록
4. **STBJ490_M0** (품절상품) → `/stbj490/selSearch` → 품절현황
5. **STMB010_M0** (시간대별 매출) → `/stmb010/selSearch` → 시간대 매출

### 검증된 패턴 (SalesAnalyzer에서 사용)

```
상위 메뉴 클릭:
  mainframe.HFrameSet00.VFrameSet00.TopFrame.form.div_topMenu.form.{PARENT_ID}:icontext

서브 메뉴 클릭:
  mainframe.HFrameSet00.VFrameSet00.TopFrame.form.pdiv_topMenu_{PARENT_ID}.form.{SUB_ID}:text

WorkFrame 접근:
  nexacro.getApplication().mainframe.HFrameSet00.VFrameSet00.FrameSet.WorkFrame.form
```

### 주의사항
- 화면 최대 7개까지만 동시 열림 → 탭 관리 필요
- `mouseover`가 아닌 실제 `mousedown→mouseup→click` MouseEvent 시퀀스 필요
- TopFrame 데이터셋은 `topForm['이름']`으로 직접 접근 (objects 배열의 typename은 undefined)
