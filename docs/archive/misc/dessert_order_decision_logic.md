# CU 편의점 디저트 발주 유지/정지 판단 로직 설계서

> 버전 1.2 | 2026년 3월 주간 데이터 기반 자동 판단 시스템 | BGF리테일 (CU) 자동발주 시스템 연동용

---

## 1. 프로젝트 개요

본 문서는 CU 편의점 디저트 카테고리의 발주 유지/정지 판단 로직을 정의합니다.
한국 편의점 디저트 시장의 특성을 반영하여, 카테고리별로 차별화된 판단 기준을 적용합니다.

### 1.1 배경

BGF리테일 보고서에 따르면, 편의점 인기 상품의 상품생애주기(PLC)가 과거 평균 22개월에서 최근 4개월로 급격히 단축되었습니다.
SNS 바이럴 기반의 "초단기 폭발 → 급락" 패턴이 보편화되면서, 빠른 판단과 대응이 필수적입니다.

### 1.2 적용 범위

- **대상**: CU 편의점 디저트 카테고리 (중분류 014) 전체 174개 상품
- **조건**: 매대 위치 고정, 행사상품 없음, 아이스디저트 없음
- **판단 방식**: 순수 판매 수치 기반 (보정 변수 최소화)

---

## 2. 카테고리 분류 체계

본사 소분류와 유통기한을 결합하여 4개 카테고리로 구분합니다.
각 카테고리는 유통기한과 폐기 리스크가 근본적으로 다르므로, 판단 주기와 기준값을 각각 다르게 적용합니다.

### 2.1 분류 규칙

1. **1차 — 소분류명(small_nm) 직접 매핑**: 냉장디저트→A, 냉장젤리,푸딩→D, 냉동디저트→D
2. **2차 — 유통기한 기반** (상온/시즌/미분류만): ≤20일→B, >20일→C
3. **3차 — 안전장치** (v1.2): C/D로 분류된 후 유통기한이 짧으면 상향 조정
   - C/D + 유통기한 ≤ 20일 → **A 상향** (냉장디저트급 엄격 관리)
   - C/D + 유통기한 21~79일 → **B 상향** (상온디저트-단기급 관리)
   - 상수: `SAFETY_SHORT_THRESHOLD=20`, `SAFETY_MEDIUM_THRESHOLD=79`
4. **폴백**: 소분류도 유통기한도 없으면 → A (보수적, 가장 엄격한 기준 적용)

| 카테고리 | 설명 | 유통기한 | 상품 수 | 폐기 리스크 | 판단 주기 |
|----------|------|----------|---------|------------|-----------|
| **A** | 냉장디저트 (생크림빵, 도넛, 케이크, 마카롱, 찰떡 등) | 2~5일 | 106개 | 최대 | 주 1회 |
| **B** | 상온디저트-단기 (크로플, 팬케이크, 휘낭시에 등) | 9~20일 | 15개 | 중간 | 2주 1회 |
| **C** | 상온디저트-장기 (와플, 양갱, 쓰베 등) | 20일 초과 | 21개 | 낮음 | 월 1회 |
| **D** | 냉장젤리/푸딩 (쁘띠첼, 젤리 등) | 80일 이상 | 30개 | 거의 없음 | 월 1회 |

## 3. 신상품 판별 로직

신상품 여부는 "첫 입고일"을 기준으로 판별하며, 3개 데이터 소스를 우선순위로 사용합니다.

### 3.1 데이터 소스 우선순위

| 우선순위 | 데이터 소스 | 설명 | 비고 |
|----------|------------|------|------|
| 1순위 | `detected_new_products.first_receiving_date` | 첫 입고일 (센터매입 확정일) | 가장 정확, 시스템 가동 이후 데이터만 존재 |
| 2순위 | `MIN(daily_sales.sale_date)` | 첫 판매일 역추정 | 입고 후 판매 전 공백 미반영 |
| 3순위 | `products.created_at` | DB 등록일 | 초기 일괄 등록 상품 주의 |

### 3.2 상품 생애주기 구분

| 구분 | 기간 | 설명 |
|------|------|------|
| **신상품** | 첫 입고 후 4주 이내 | 보호 기간 -- 고객 인지 기간 필요. 단, 급락 경고 태그 부착 |
| **성장/하락기** | 4주 ~ 8주 | 핵심 판단 구간 -- 유지/정지 판단이 이루어지는 구간 |
| **정착기** | 8주 초과 | 안정 상품 vs 소멸 상품 분리 |

## 4. 카테고리별 판단 기준

### 4.1 카테고리 A: 냉장디저트 (주 1회 판단)

폐기 리스크가 가장 높은 카테고리로, 가장 엄격한 기준을 적용합니다.

| 생애주기 | 판단 기준 | 결과 |
|----------|----------|------|
| 신상품 (4주 이내) | 무조건 유지 | 단, 주간 판매 하락률 50% 이상시 "급락 경고" 태그 |
| 성장/하락기 (4~8주) | 주간 판매율 50% 미만 2주 연속 | 정지 권고 |
| 정착기 (8주+) | 주간 판매량 < 카테고리 평균의 30% | 정지 권고 |
| **전 구간 공통** | **폐기금액 > 판매금액** | **즉시 정지 권고** |

### 4.2 카테고리 B: 상온디저트-단기 (2주 1회 판단)

| 생애주기 | 판단 기준 | 결과 |
|----------|----------|------|
| 신상품 (3주 이내) | 보호 기간 -- 판단 유예 | 추세 모니터링만 수행 |
| 이후 | 판매율 40% 미만 3주 연속 | 정지 권고 |

### 4.3 카테고리 C: 상온디저트-장기 (월 1회 판단)

| 생애주기 | 판단 기준 | 결과 |
|----------|----------|------|
| 신상품 (4주 이내) | 보호 기간 -- 판단 유예 | 추세 모니터링만 수행 |
| 이후 | 월간 판매량 < 카테고리 평균의 20% | 정지 권고 (매대 공간 효율 관점) |

### 4.4 카테고리 D: 냉장젤리/푸딩 (월 1회 판단)

| 판단 기준 | 결과 | 비고 |
|----------|------|------|
| 월간 판매량 0개가 2개월 연속 | 정지 권고 | 유통기한이 길어 폐기 리스크 거의 없음 |

## 5. 전체 판단 흐름

```
1. 상품 입력
   → 소분류(냉장디저트/상온디저트/냉장젤리푸딩) + 유통기한으로 카테고리 A~D 분류

2. 첫 입고일 기준으로 생애주기 판별 (신상품 / 성장하락기 / 정착기)

3. 카테고리별 주기에 맞춰 판매 데이터 집계 (판매수량, 폐기수량, 판매율)

4. 판매율 + 폐기금액 + 추세 계산

5. 유지 / 관찰 / 정지 권고 분류

6. 사람 최종 확인 후 발주 상태 변경
```

## 6. 구현 방향

### 6.1 파일 및 모듈 구성

| 파일/모듈 | 역할 | 설명 |
|-----------|------|------|
| `dessert_category.py` | 카테고리 분류기 | 소분류 + 유통기한으로 A~D 카테고리 분류 |
| `dessert_lifecycle.py` | 생애주기 판별 | 3개 데이터 소스 우선순위로 첫 입고일 판별 → 생애주기 구분 |
| `dessert_judge.py` | 판단 엔진 | 카테고리 + 생애주기별 판매율/폐기금액/추세 기반 판단 |
| `dessert_scheduler.py` | 스케줄러 | 카테고리별 판단 주기에 맞춰 자동 실행 (주1회 / 2주1회 / 월1회) |
| `dessert_report.py` | 결과 리포트 | 유지/관찰/정지권고 리스트 출력 → 사람 최종 확인용 |

### 6.2 데이터 흐름

```
[ products 테이블 + detected_new_products 테이블 ]
  → 카테고리 분류 + 생애주기 판별
  → [ daily_sales 테이블 ]
  → 판매율 / 폐기금액 / 추세 계산
  → 판단 엔진
  → 유지 / 관찰 / 정지권고 결과
  → 사람 확인
  → 발주 상태 반영
```

### 6.3 기존 시스템 연동 포인트

- `detected_new_products` 테이블의 `lifecycle_status` 필드를 활용하여 신상품 모니터링 결과와 연계
- `daily_sales` 테이블에서 `mid_cd = '014'` (디저트) 필터링 후 집계
- 판단 결과를 별도 테이블(`dessert_decisions`)에 기록하여 이력 추적
- 기존 ML 예측 모델의 예측값과 실제 판매량 괴리도 보조 지표로 활용 가능

---

## 7. 향후 확장 방향

- **판단 기준값 자동 튜닝**: 실제 운영 데이터 축적 후, 최적 판매율 기준값을 매장별로 자동 조정
- **다매장 확장**: 매장별 고객층 차이를 반영한 카테고리별 기준값 차별화
- **SNS 트렌드 연동**: 외부 바이럴 지표를 신상품 보호 기간 연장/단축 판단에 활용
- **디저트 외 카테고리 확장**: 동일 로직을 간편식, 음료 등 다른 카테고리에도 적용
