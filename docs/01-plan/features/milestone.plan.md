---

## 1. 문제 정의

### 현상
이슈체인이 **운영 이상 감지**(소방)에는 잘 동작하지만, 프로젝트가 **원하는 방향으로 진행되고 있는지**(나침반) 판단할 수 없다.

- PLANNED 7건이 모두 P2~P3으로 실행 순서 불분명
- "3단계: 발주 품질 개선"의 완료 조건이 정의되지 않음
- 이슈체인 자동 감지는 악화만 감지, 목표 달성 여부는 감지 안 함

### 근본 원인
이슈체인 시스템(`ops_anomaly.py`)이 **비율 기반 악화 감지**(`mae_7d > mae_14d * 1.2`)만 구현. **절대 목표 도달 감지**(`mae_7d < target`)는 존재하지 않음.

### 해결 방향
1. 3단계 완료를 판단하는 **측정 가능한 목표 지표** 정의
2. 각 PLANNED 이슈가 어떤 목표에 기여하는지 **매핑**
3. 목표 도달을 감지하는 **양성 지표 감지** 추가 (선택)

## 2. 목표 지표 정의

> 아래 목표값은 현재 운영 공식 기반 초안. 2주 실측 후 조정.

### 핵심 KPI (4개 — 모두 충족 시 3단계 완료)

| # | 지표 | 측정 공식 | 목표 | 현재 감지 방식 |
|---|------|----------|------|--------------|
| K1 | 예측 안정성 | `mae_7d / mae_14d` (4매장 평균) | < 1.05 (5% 이내 변동) | 악화만 감지 (> 1.2) |
| K2 | 폐기율 | `waste_qty / (waste_qty + sales_qty)` 30일 | food < 3%, 전체 < 2% | 악화만 감지 (> 1.5x) |
| K3 | 발주 실패율 | `fail_count_7d / total_order_7d` | < 5% | 악화만 감지 (> 1.5x) |
| K4 | 데이터 무결성 | `max(consecutive_anomaly_days)` 전 체크 | < 3일 | 악화만 감지 (> 7일) |

### 보조 KPI (3개 — 참고용, 완료 조건 아님)

| # | 지표 | 측정 공식 | 목표 |
|---|------|----------|------|
| S1 | MAE 절대값 | `AVG(ABS(predicted - actual))` 7일 | food < 1.5, 비food < 1.0 |
| S2 | GHOST_STOCK 오탐률 | `false_positive / total_detect` 14일 | < 5% (자동화 승격 기준) |
| S3 | 신상품 종합점수 | `intro_score + ds_score` | >= 95 (지원금 160K) |

### 4단계 예비 정의: 수익성 최적화

> 3단계가 "발주가 정확하고 안정적인가"라면, 4단계는 "발주가 돈을 벌고 있는가".

**방향**:
- 폐기 비용 최소화 (단순 폐기율이 아닌 원가 기반)
- 품절 기회손실 측정 + 최소화
- 신상품 도입 지원금 극대화 (종합점수 95+)
- 카테고리별 마진율 기반 발주 우선순위

**예비 KPI (3단계 완료 시점에 확정)**:

| # | 지표 | 방향 |
|---|------|------|
| K5 | 월간 폐기 원가 | 전월 대비 감소 |
| K6 | 품절 기회손실 추정액 | 측정 체계 구축 → 감소 |
| K7 | 신상품 종합점수 | >= 95 (지원금 160K) |
| K8 | 카테고리별 발주 마진 기여 | 측정 체계 구축 |

**선행 조건**: 3단계 K1~K4 전부 ACHIEVED 2주 연속

> 4단계 상세 계획은 3단계 완료 시점에 별도 Plan 문서로 작성한다.
