## 개요

**기능명**: 운영 지표 → 이슈 자동 등록 파이프라인
**목적**: 운영 데이터에서 이상 패턴을 자동 감지 → 이슈 체인 `[PLANNED]` 자동 등록 → 카카오 알림

### AS-IS → TO-BE
```
AS-IS: 사용자 요청 → 이슈 등록 → 작업 → 검증
TO-BE: 운영 로그 자동 분석 → 이상 감지 → 이슈 자동 등록 → 카카오 알림
                                                    ↓
                              SessionStart hook → Claude 자동 인지
```

## 모니터링 대상 지표 (5개)

| # | 지표 | 소스 테이블 | 감지 조건 | 기본 우선순위 | P1 승격 조건 |
|---|------|-----------|----------|:---:|------|
| 1 | 예측 정확도 하락 | eval_outcomes | 카테고리별 7d MAE > 14d MAE × 1.2 | P2 | 3개+ 카테고리 동시 |
| 2 | 발주 실패 급증 | order_fail_reasons | 최근 7d 실패 > 이전 7d × 1.5 | **P1** | 항상 |
| 3 | 폐기율 상승 | waste_slips + daily_sales | 카테고리별 7d 폐기율 > 30d 평균 × 1.5 | P2 | 푸드(001~005) |
| 4 | 수집 연속 실패 | collection_logs | 동일 유형 3일 연속 실패 | **P1** | 항상 |
| 5 | 자전 시스템 미해결 | integrity_checks | 특정 check_name 7일 연속 anomaly > 0 | P2 | 14일 연속 |

## 아키텍처

```
[매일 23:55 스케줄]
       │
       ▼
 OpsIssueDetector (Application — 오케스트레이션)
       │
       ├── OpsMetrics (Analysis): 5개 지표 DB 수집
       │     └─ 기존 테이블만 조회 (새 테이블 불필요)
       │
       ├── OpsAnomaly (Domain): 임계값 비교 → 이상 판정
       │     └─ 순수 함수, I/O 없음
       │     └─ 중복 방지: 기존 [PLANNED]/[OPEN]/[WATCHING] 동일 제목 존재 시 스킵
       │     └─ 14일 쿨다운: 최근 [RESOLVED] 동일 패턴이면 스킵
       │
       ├── IssueChainWriter (Infrastructure): .md 파일 안전 삽입
       │     └─ 마지막 --- 구분자 위에 [PLANNED] 블록 추가
       │     └─ **감지**: 자동 감지 (날짜) 태그로 수동 등록과 구분
       │
       └── sync_issue_table.py + KakaoNotifier
             └─ CLAUDE.md 테이블 자동 갱신 + 카카오 알림
```

## 파일 구조

| 파일 | 위치 | 계층 | 역할 |
|------|------|------|------|
| `ops_metrics.py` | `src/analysis/` | Analysis | 5개 지표 DB 조회 → 집계 dict 반환 |
| `ops_anomaly.py` | `src/domain/` | Domain | 순수 함수: 임계값 비교 → anomaly list 반환 |
| `issue_chain_writer.py` | `src/infrastructure/` | Infrastructure | .md 파일 읽기/파싱/삽입 (I/O) |
| `ops_issue_detector.py` | `src/application/services/` | Application | 오케스트레이션: 수집→판정→등록→알림 |

## 이슈 등록 형식

```markdown
## [PLANNED] {카테고리} 예측 정확도 하락 조사 (P2)

**목표**: {카테고리} 7일 MAE {값}이 14일 평균 {값} 대비 {비율}% 악화 원인 조사
**동기**: 자동 감지 (2026-04-05)
**선행조건**: 없음
**예상 영향**: prediction/ 관련 파일
```

## 중복 방지 로직

1. 이슈 체인 파일에서 `[PLANNED]`/`[OPEN]`/`[WATCHING]` 제목 추출
2. 새 이슈 제목과 유사도 비교 (핵심 키워드 매칭)
3. 이미 존재하면 등록 스킵 + 로그만 기록
4. 최근 14일 내 `[RESOLVED]`로 전환된 동일 패턴이면 스킵 (재발 확인 기간)

## 우선순위 자동 결정

| 지표 | 기본 | P1 승격 조건 |
|------|:---:|------|

| 수집 연속 실패 | **P1** | 항상 긴급 (데이터 없으면 전체 중단) |
| 예측 정확도 하락 | P2 | 3개+ 카테고리 동시 하락 |
| 폐기율 상승 | P2 | 푸드(001~005) 카테고리 |
| 자전 시스템 미해결 | P2 | 14일 연속 |

## 성공 기준

1. 자동 등록된 이슈가 사용자 요청보다 **먼저** 문제를 포착하는 사례 1건 이상
2. 오탐률(false positive) **30% 이하**
3. 사용자 일일 로그 확인 시간 감소
