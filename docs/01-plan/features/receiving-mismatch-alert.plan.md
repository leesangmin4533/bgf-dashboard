# Plan: 발주-입고 불일치 감지 및 알림

## 1. 개요

### 목표
발주(order_tracking) vs 센터매입(receiving_history) 대조로 불일치 감지:
- 미입고 (발주O 센터X): 폐기 알림 제외 + "미입고" 알림
- 미발주 입고 (발주X 센터O): 별도 알림
- 수량 불일치: 검수 확인 요청

### 핵심 변수: 발주요일별 입고 리드타임

| 발주요일 패턴 | 건수 | 최대 리드타임 | 비고 |
|-------------|------|------------|------|
| 일월화수목금토 (매일) | 2,016 | **1~2일** | 푸드 대부분 |
| 월화수목금토 (주6일) | 2,982 | **2일** | 비푸드 대부분 |
| 화목토 (주3일) | 1,220 | **3일** | 스낵/비스켓 |
| 월수금 (주3일) | 1,125 | **3일** | |
| 수금 (주2일) | 27 | **4일** | |

**푸드(매일)는 1~2일 내 입고 확인, 비푸드는 상품별 발주요일에 따라 최대 4일 대기 후 판정.**

## 2. 대조 로직

### 대조 시점
- 매일 07:00 수집 완료 후 (07:20경)
- 센터매입 수집(Phase 1.1) + daily_sales 수집(Phase 1.0) 완료 상태

### 대조 공식

```
각 발주건(order_tracking)에 대해:
  발주일 = order_date
  상품별 리드타임 = orderable_day 기반 최대 입고 대기일
  대기 마감일 = 발주일 + 리드타임 + 1일 (여유)

  if 현재일 > 대기 마감일:
    센터매입에 해당 item_cd 있는지 확인
    → 없으면: "미입고" 확정
    → 있지만 수량 다름: "수량 불일치"
  else:
    → 아직 대기 중 (판정 보류)
```

### 리드타임 계산

```python
def calc_max_lead_days(orderable_day: str) -> int:
    """발주가능요일 기반 최대 입고 리드타임 계산

    발주 → 다음 발주가능일까지 최대 간격 + 입고 1일
    """
    if not orderable_day:
        return 3  # 기본값

    days = ['일','월','화','수','목','금','토']
    available = [i for i, d in enumerate(days) if d in orderable_day]

    if len(available) >= 7:
        return 2  # 매일 → 1~2일

    # 최대 간격 계산 (순환)
    max_gap = 0
    for i in range(len(available)):
        next_i = (i + 1) % len(available)
        gap = (available[next_i] - available[i]) % 7
        max_gap = max(max_gap, gap)

    return max_gap + 1  # 간격 + 입고 1일
```

## 3. 불일치 유형별 처리

| 유형 | 조건 | 알림 | 폐기 알림 영향 |
|------|------|------|-------------|
| **미입고** | OT에 있고, 대기 마감 초과, 센터매입에 없음 | "미입고 N건 — 본사 확인 필요" | OT에서 **제외** |
| **수량 불일치** | OT 수량 ≠ 센터매입 수량 | "수량 차이 — 검수 확인" | 센터매입 수량 기준 |
| **미발주 입고** | 센터매입에 있고, OT에 없음 | "미발주 입고 N건" | 배치 생성됨 (정상) |
| **대기 중** | 발주 후 리드타임 이내 | 알림 없음 | OT에서 **제외** (아직 미입고) |

## 4. OT 폐기 알림 영향

**핵심 수정**: `get_items_expiring_at()`에서 OT 조회 시 `status='arrived'`만 포함

```python
# Before: ordered(미입고)도 포함 → 오보
WHERE status IN ('ordered', 'arrived')

# After: 입고 확인된 것만
WHERE status = 'arrived'
```

미입고 상품은 별도 "미입고 알림"으로 처리.

## 7. 리스크

| 리스크 | 대응 |
|--------|------|
| 센터매입 수집 시점 vs 실제 입고 시점 차이 | 리드타임에 +1일 여유 |
